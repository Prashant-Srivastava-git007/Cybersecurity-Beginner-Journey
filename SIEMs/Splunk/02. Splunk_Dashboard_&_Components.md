## Splunk Core Components

### 1. Splunk Forwarder
The **Splunk Forwarder** is a lightweight agent installed on systems where data is generated.  
Its main role is to **collect data and send it to Splunk** with minimal impact on system performance.

**Common data sources:**
- Web servers (web traffic logs)
- Windows systems (Event Logs, PowerShell, Sysmon)
- Linux systems (system and application logs)
- Databases (connections, queries, errors)

---

### 2. Splunk Indexer
The **Indexer** receives data from forwarders and processes it.

**What it does:**
- Parses and normalizes raw data
- Converts data into searchable events (fieldâ€“value format)
- Stores data efficiently for fast searching

The indexer is the backbone of data storage and search performance.

---

### 3. Splunk Search Head
The **Search Head** is the user interface where searches are performed.

**Key responsibilities:**
- Provides access to the Search & Reporting app
- Accepts user searches written in SPL (Search Processing Language)
- Sends queries to indexers and displays results

## Splunk Dashboard Overview

![Splunk_Dashboard](Splunk/Asset/Splunk-Dashboard.png)

### 1. Apps
Apps extend Splunk for specific use cases.

**Key functions:**
- View installed apps
- Install new apps from Splunkbase
- Manage app access and permissions

Examples include Search & Reporting, Security apps, and infrastructure monitoring apps.

---

### 2. Bookmarks
Bookmarks allow quick access to frequently used items.

**Use cases:**
- Save dashboards, searches, or reports
- Organize personal or shared shortcuts
- Improve analyst productivity

---

### 3. Dashboards
Dashboards display visual representations of data.

**Features:**
- Show charts, tables, and panels
- Support real-time and historical views
- Help monitor trends and system health

---

### 4. Search History
This section keeps a record of previous searches.

**Benefits:**
- Reuse and refine past queries
- Validate previous investigations
- Improve SPL efficiency

---

### 5. Recently Viewed
Shows content accessed recently.

**Advantages:**
- Quickly return to active work
- Avoid searching again for the same objects
- Maintain workflow continuity

---

### 6. Created by You
Lists content created by the logged-in user.

**Purpose:**
- Track personal dashboards and searches
- Manage ownership and permissions
- Maintain organized analysis work

---

### 7. Shared with You
Displays content shared by others.

**Use cases:**
- Access team dashboards
- Collaborate across teams
- Follow standardized monitoring views

---

## Settings Overview

### A. Add Data
Used to onboard data into Splunk.

**Capabilities:**
- Upload log files
- Configure live data inputs
- Assign source types and indexes

---

### B. Knowledge
Controls how data is structured and enriched.

**Includes:**
- Saved searches and alerts
- Field extractions and tags
- Lookups and data models

---

### C. Data
Manages data collection and storage.

**Key areas:**
- Data inputs
- Indexes and retention
- Forwarding and receiving
- Source types and ingest actions

---

### D. Distributed Environment
Used in large or multi-system deployments.

**Supports:**
- Indexer and search head clustering
- Distributed and federated search
- Forwarder management

---

### E. System
Controls internal Splunk operations.

**Includes:**
- Server settings
- Licensing
- Health and diagnostics
- Performance management

---

### F. Users and Authentication
Manages access control.

**Functions:**
- User and role management
- Authentication methods (local, LDAP, SAML)
- Token and permission control

This ensures secure, role-based access to Splunk.
